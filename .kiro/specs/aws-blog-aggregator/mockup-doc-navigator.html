AWS Lambda Resource-based Policies
Learn how to use resource-based policies to grant permissions to other AWS services and accounts
Overview
AWS Lambda supports resource-based permissions policies for Lambda functions and layers. Resource-based policies let you grant usage permission to other AWS accounts or organizations on a per-resource basis. You also use a resource-based policy to allow an AWS service to invoke your function.
3.2 Resource-based Policy Statements
🔍 AI FOUND THIS
To give other accounts and AWS services permission to use your Lambda function, use the add-permission command. For example, to grant Amazon S3 permission to invoke your function, use the following command with the --principal s3.amazonaws.com parameter and specify the S3 bucket ARN as the --source-arn.
AWS CLI # Grant S3 permission to invoke Lambda function
aws lambda add-permission \
  --function-name my-function \
  --principal s3.amazonaws.com \
  --statement-id s3-invoke \
  --action lambda:InvokeFunction \
  --source-arn arn:aws:s3:::my-bucket \
  --source-account 123456789012
📘 Note: Each Lambda function can have a single resource-based policy with up to 20 KB of policy statements.
3.3 Cross-Account Access
To grant cross-account access, specify the full account ID in the principal. The following example grants account 210987654321 permission to invoke the function.
AWS CLI aws lambda add-permission \
  --function-name my-function \
  --principal 210987654321 \
  --statement-id cross-account \
  --action lambda:InvokeFunction
⚠️ Important: When you grant cross-account access, the other account must also have an IAM policy that allows the user to call the Lambda API.
3.4 Viewing Resource-based Policies
To view a function's resource-based policy, use the get-policy command. This returns the JSON policy document attached to the specified function.
AWS CLI aws lambda get-policy \
  --function-name my-function
The output includes the policy document and a revision ID. You can use the revision ID to update the policy with the add-permission command.

📍 Direct Answer

To give Lambda permission to read from S3, you need to use the add-permission CLI command with s3.amazonaws.com as the principal. Specify your S3 bucket ARN as the source.
📄 Source: Section 3.2 - Resource-based Policies (highlighted in doc →)

💻 Code Example

# Grant S3 read permission
aws lambda add-permission \
  --function-name my-function \
  --principal s3.amazonaws.com \
  --statement-id s3-read \
  --action lambda:InvokeFunction \
  --source-arn arn:aws:s3:::YOUR-BUCKET

🔐 Also Needed: IAM Execution Role

Your Lambda function also needs an execution role with S3 read permissions:
// IAM Policy for Lambda
{
  "Effect": "Allow",
  "Action": [
    "s3:GetObject",
    "s3:ListBucket"
  ],
  "Resource": "arn:aws:s3:::YOUR-BUCKET/*"
}
